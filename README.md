## Как пользоваться

### Установка

1. Скачайте **MangaDownloader.exe** из раздела [Releases](../../releases).
2. Убедитесь, что на компьютере установлен **Google Chrome** (нужен для авторизации на сайте).
3. Запустите `MangaDownloader.exe` — установка не требуется.

### Первый запуск (режим браузера)

1. Нажмите **«Открыть сайт и начать»** — откроется Chrome с сайтом com-x.life.
2. **Войдите в свой аккаунт** на сайте. Приложение ждёт до 5 минут и автоматически сохранит cookies.
3. Перейдите на страницу нужной манги — приложение обнаружит её и заменит кнопку «Отслеживать» на **«Скачать»**.
4. Нажмите **«Скачать»** на странице манги.
5. В появившемся диалоге выберите:
   - **Все главы** — скачать всё с нуля в новый архив.
   - **Диапазон глав** — указать конкретные главы (от / до).
   - **Режим архива** — создать новый CBZ или дополнить существующий.
6. Нажмите **«Скачать»** в диалоге — начнётся загрузка.
7. По завершении CBZ-файл будет готов. Кнопка **«Открыть папку»** откроет папку с результатом в проводнике.

### Скачивание из библиотеки

После первого скачивания манга добавляется в **библиотеку** (список в главном окне). Далее:

- Приложение **автоматически проверяет** наличие новых глав каждые 5 минут.
- Если есть новые главы, рядом с названием появится бейдж **+N**.
- Нажмите **«Скачать»** напротив нужной манги — откроется диалог с предложением докачать только новые главы.
- Скачивание из библиотеки работает **без браузера** — используются сохранённые cookies.

### Управление библиотекой

- **Кнопка «✕»** — удаляет мангу из библиотеки (с подтверждением). Также удаляет CBZ-файл.
- Библиотека сортируется по дате последнего скачивания (новые сверху).

### FAQ

**Q: Скачивание из библиотеки не работает, пишет про cookies.**
A: Cookies авторизации устарели. Нажмите «Открыть сайт и начать», войдите в аккаунт заново — cookies обновятся автоматически.

**Q: Все методы загрузки не сработали.**
A: Убедитесь, что Google Chrome установлен и обновлён до последней версии. Попробуйте заново через «Открыть сайт и начать» для обновления сессии.

**Q: Где хранятся скачанные файлы?**
A: В папке `output/` рядом с EXE-файлом. Формат — CBZ (ZIP-архив с изображениями, открывается любой читалкой комиксов).

**Q: Можно ли докачать новые главы в существующий архив?**
A: Да. При скачивании из библиотеки выберите «Диапазон глав» и режим «Дополнить существующий». Нумерация страниц продолжится автоматически.

**Q: Приложение не запускается / вылетает при старте.**
A: Попробуйте запустить EXE от имени администратора. Также убедитесь, что антивирус не блокирует приложение.

## Для контрибьюторов

### Архитектура проекта

Приложение построено по принципу разделения ответственности:

```
┌─────────────────────────────────────────────────────────┐
│                    GUI (PyQt5)                           │
│  DownloaderApp — главное окно, библиотека, логи          │
│  ChapterSelectDialog — выбор глав и режима               │
├─────────────────────────────────────────────────────────┤
│              Фоновые потоки (QThread)                     │
│  ChapterWorker — браузер, мониторинг, скачивание, CBZ    │
│  UpdateChecker — проверка новых глав (ThreadPoolExecutor) │
├─────────────────────────────────────────────────────────┤
│                 Бизнес-логика                             │
│  MangaParser — парсинг HTML, извлечение window.__DATA__  │
│  FallbackDownloader — оркестрация цепочки загрузчиков    │
│  CookieManager — управление cookies                      │
│  DownloadHistory — JSON-хранилище библиотеки             │
├─────────────────────────────────────────────────────────┤
│               Загрузчики (downloaders/)                   │
│  CurlCffiDownloader → CloudscraperDownloader →           │
│  → SeleniumRecoveryDownloader                            │
└─────────────────────────────────────────────────────────┘
```

Ключевые принципы:
- **GUI не содержит бизнес-логики** — вся работа делегируется `ChapterWorker`.
- **Общение GUI ↔ Worker только через Qt-сигналы** — потокобезопасность.
- **Загрузчики следуют паттерну Template Method** — общая логика в `BaseDownloader`, подклассы реализуют `_api_request` и `_download_file`.
- **Fallback-цепочка** — если один метод загрузки не сработал, автоматически пробуется следующий.

### Как работает скачивание

Полный цикл скачивания манги:

#### 1. Авторизация

`ChapterWorker` открывает Chrome через Selenium и переходит на com-x.life. Если есть сохранённые cookies — пробует восстановить сессию. Если нет или они устарели — ждёт ручного входа пользователя (до 5 минут). После успешной авторизации cookies сохраняются в `comx_life_cookies_v3.json`.

#### 2. Обнаружение манги

Worker мониторит текущий URL браузера в цикле (каждые 500 мс). Когда пользователь открывает страницу манги (URL содержит `.html`):

- `MangaParser` загружает HTML страницы через `curl_cffi`.
- Из HTML извлекается JavaScript-объект `window.__DATA__` с помощью регулярного выражения.
- Из `__DATA__` парсятся: название, `news_id`, список глав с их `id` и `title`.
- На странице кнопка «Отслеживать» заменяется на «Скачать» через JS-инъекцию.

#### 3. Запрос на скачивание

Когда пользователь нажимает «Скачать», URL меняется на `…/download`. Worker это обнаруживает, закрывает браузер и показывает диалог выбора глав.

#### 4. Загрузка глав

Для каждой главы из выбранного диапазона:

1. **API-запрос** — `POST` на `https://com-x.life/engine/ajax/controller.php?mod=api&action=chapters/download` с параметрами `chapter_id` и `news_id`.
2. **Получение URL** — из JSON-ответа извлекается поле `data` с URL ZIP-файла.
3. **Скачивание ZIP** — файл скачивается и валидируется как корректный ZIP.
4. **Задержка** — между главами пауза 1.5 секунды (`REQUEST_DELAY`).

Если метод загрузки не сработал — автоматически пробуется следующий (см. [Система fallback-загрузчиков](#система-fallback-загрузчиков)).

#### 5. Сборка CBZ

После скачивания всех глав:

1. Все ZIP-файлы глав обрабатываются по порядку.
2. Из каждого ZIP извлекаются изображения (`.jpg`, `.png`, `.gif`, `.webp`, `.bmp`).
3. Изображения переименовываются в последовательные номера: `000001.jpg`, `000002.png`, ...
4. Все изображения упаковываются в один CBZ-файл.
5. В режиме «дополнить» нумерация продолжается с последней страницы существующего архива.
6. Временные файлы удаляются.

### Система fallback-загрузчиков

`FallbackDownloader` пробует три метода по цепочке. Если первый успешен — остальные не вызываются.

```
CurlCffiDownloader          # Приоритет 1: быстрый, эмулирует TLS Chrome
    ↓ (при ошибке)
CloudscraperDownloader      # Приоритет 2: обход Cloudflare challenge
    ↓ (при ошибке)
SeleniumRecoveryDownloader  # Приоритет 3: открывает Chrome, обновляет cookies
```

| Загрузчик | Как работает | Когда нужен |
|-----------|-------------|-------------|
| `CurlCffiDownloader` | HTTP-запросы через `curl_cffi` с `impersonate="chrome"` — эмулирует TLS-отпечаток Chrome | Основной метод, работает в большинстве случаев |
| `CloudscraperDownloader` | Использует `cloudscraper` для автоматического решения Cloudflare JS-challenge | Когда curl_cffi получает 403 от Cloudflare |
| `SeleniumRecoveryDownloader` | Открывает реальный Chrome, обновляет cookies через браузер, затем повторяет запрос через `curl_cffi` | Когда cookies устарели и нужна полная перезагрузка сессии |

Все загрузчики наследуют `BaseDownloader` и реализуют паттерн **Template Method**:

```python
class BaseDownloader(ABC):
    def download(self, chapter_id, news_id, zip_path, title) -> bool:
        # 1. Вызвать _api_request (абстрактный)
        # 2. Извлечь URL из ответа
        # 3. Вызвать _download_file (абстрактный)
        # 4. Валидировать ZIP
        ...

    @abstractmethod
    def _api_request(self, chapter_id, news_id) -> dict: ...

    @abstractmethod
    def _download_file(self, url, dest) -> None: ...
```

Исключение — `SeleniumRecoveryDownloader` полностью переопределяет `download()`, так как его логика принципиально отличается (нужно сначала открыть браузер).

### GUI и потоки

Приложение использует **три типа потоков**:

| Поток | Класс | Назначение |
|-------|-------|-----------|
| Главный (UI) | `DownloaderApp` | Отрисовка интерфейса, обработка событий пользователя |
| Рабочий | `ChapterWorker` (QThread) | Браузер, парсинг, скачивание, сборка CBZ |
| Проверка обновлений | `UpdateChecker` (QThread + ThreadPoolExecutor) | Параллельная проверка новых глав |

**Связь между потоками** — только через Qt-сигналы:

```
ChapterWorker                         DownloaderApp
─────────────                         ─────────────
log(str)                ──────►       _append_log()
chapter_progress(i,n,t) ──────►       _on_chapter_progress()
manga_info_ready(...)   ──────►       _on_manga_info_ready() → показ диалога
chapters_found(...)     ──────►       _on_chapters_found()
cbz_ready(path)         ──────►       _on_cbz_ready()
download_complete_info  ──────►       _on_download_complete_info() → обновление истории
finished_ok(bool)       ──────►       _on_finished() → разблокировка UI
```

Для синхронизации «воркер ждёт подтверждения из UI» используются `threading.Event`:
- `_confirm_event` — воркер блокируется на `.wait()`, пока пользователь не подтвердит скачивание в диалоге.
- `_cancel_event` — отмена скачивания из UI.

### Библиотека и история

`DownloadHistory` хранит данные в `manga_history.json`:

```json
{
  "version": 1,
  "manga": {
    "https://com-x.life/12345-manga-name.html": {
      "title": "Название манги",
      "url": "https://com-x.life/12345-manga-name.html",
      "news_id": "12345",
      "downloaded_chapters": [1, 2, 3, 4, 5],
      "last_chapter_downloaded": 5,
      "last_known_total": 10,
      "cbz_path": "C:/path/to/output/Название_манги.cbz",
      "last_download_date": "2026-02-17T12:00:00",
      "download_count": 2
    }
  }
}
```

Ключевые поля:
- `downloaded_chapters` — список индексов скачанных глав (для определения пропусков).
- `last_known_total` — общее количество глав на сайте (обновляется `UpdateChecker`).
- `cbz_path` — абсолютный путь к CBZ-файлу (для режима «дополнить»).

При повторном скачивании `upsert()` мержит списки глав (объединение множеств).

### Проверка обновлений

`UpdateChecker` запускается:
- При старте приложения.
- Каждые 5 минут (`QTimer`).
- После завершения скачивания.

Для каждой манги в библиотеке:
1. `MangaParser.fetch_quick()` загружает страницу с коротким таймаутом (10 сек).
2. Парсит `window.__DATA__` и считает количество глав.
3. Сравнивает с `last_chapter_downloaded` — разница = новые главы.
4. Результат отправляется через сигнал `result(url, total)`.

Проверки выполняются параллельно через `ThreadPoolExecutor` (до 3 потоков).

### Конфигурация

Все настройки собраны в `config.py`:

| Константа | Значение | Описание |
|-----------|----------|----------|
| `BASE_URL` | `https://com-x.life` | Базовый URL сайта |
| `API_URL` | `.../controller.php?mod=api&action=chapters/download` | Endpoint API скачивания |
| `HTTP_TIMEOUT` | 30 сек | Таймаут API-запросов |
| `DOWNLOAD_TIMEOUT` | 60 сек | Таймаут скачивания файлов |
| `LOGIN_WAIT_TIMEOUT` | 300 сек | Ожидание ручной авторизации |
| `REQUEST_DELAY` | 1.5 сек | Пауза между скачиванием глав |
| `FALLBACK_DELAY` | 1 сек | Пауза между fallback-попытками |
| `POLL_INTERVAL` | 0.5 сек | Интервал мониторинга URL в браузере |
| `IMAGE_EXTENSIONS` | `.jpg .jpeg .png .gif .webp .bmp` | Допустимые форматы изображений |

Пути к файлам вычисляются относительно корня проекта (`BASE_DIR`).

### Добавление нового загрузчика

Чтобы добавить новый метод загрузки:

1. Создайте класс, наследующий `BaseDownloader`:

```python
class MyDownloader(BaseDownloader):
    name = "my_method"

    def _api_request(self, chapter_id, news_id) -> dict:
        # POST к API, вернуть JSON
        ...

    def _download_file(self, url, dest) -> None:
        # Скачать файл по URL в dest
        ...
```

2. Добавьте его в цепочку в `FallbackDownloader.__init__()`:

```python
self._downloaders = [
    CurlCffiDownloader(...),
    CloudscraperDownloader(...),
    MyDownloader(...),          # ← новый метод
    SeleniumRecoveryDownloader(...),
]
```

3. Если загрузчику нужны дополнительные зависимости — добавьте их в `pyproject.toml` и `requirements.txt`.
